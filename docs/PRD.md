## 📋 프로젝트 개요

### 프로젝트명
**DevNotes** - 개발 노트 & 학습 자료 사이트

### 목적
Notion을 CMS로 활용한 개발자의 개인 학습 자료 저장소 및 공유 플랫폼입니다. Notion에서 작성한 개발 노트, 학습 자료, 코드 스니펫을 자동으로 웹사이트에 반영하여 언제 어디서나 접근 가능하게 합니다.

### CMS 선택 이유
- **비개발자도 관리 가능**: Notion 인터페이스만으로 콘텐츠 작성 및 관리
- **실시간 업데이트**: Notion에서 수정하면 즉시 웹사이트에 반영
- **무료 데이터베이스**: 별도 데이터베이스 구축 불필요
- **풍부한 포맷 지원**: 텍스트, 이미지, 코드 블록, 테이블 등 다양한 콘텐츠 형식 지원
- **협업 용이**: Notion 링크 공유로 쉬운 협업

## 🎯 주요 기능

### 1. 학습 자료 목록 조회
- Notion 데이터베이스에서 학습 자료 목록 가져오기
- 카드 형식의 그리드 또는 리스트 뷰로 표시
- 최신 순, 인기 순 정렬 기능

### 2. 자료 상세 페이지
- 개별 학습 자료의 전체 내용 표시
- Notion 페이지 콘텐츠를 HTML로 변환하여 렌더링
- 작성일, 카테고리, 태그 정보 표시
- 목차(Table of Contents) 자동 생성

### 3. 카테고리별 필터링
- 프로그래밍 언어별 (JavaScript, Python, Go 등)
- 주제별 (알고리즘, 웹개발, 데이터베이스 등)
- 난이도별 (초급, 중급, 고급)
- 시간별 (최신, 인기, 조회수)

### 4. 검색 기능
- 제목, 내용, 태그 기반 전문 검색
- 검색 결과 하이라이팅
- 검색어 자동완성 (선택 사항)

### 5. 반응형 디자인
- 모바일, 태블릿, 데스크톱 최적화
- 다크모드 지원
- 접근성 준수 (a11y)

### 6. 조회수 및 인기도 추적
- 각 자료별 조회수 기록
- 인기 자료 섹션
- 사용자 선호도 시각화 (선택 사항)

## 📊 Notion 데이터베이스 구조

### 데이터베이스명: **LearningNotes** (또는 **DevNotes**)

| 필드명 | 타입 | 설명 |
|--------|------|------|
| **Title** | Title | 학습 자료 제목 |
| **Category** | Select | 카테고리 (JavaScript, Python, TypeScript, React, etc.) |
| **Tags** | Multi-select | 세부 태그 (배열, 객체, 함수형프로그래밍, etc.) |
| **Difficulty** | Select | 난이도 (초급, 중급, 고급) |
| **Published Date** | Date | 발행 일자 |
| **Updated Date** | Date | 마지막 수정 일자 |
| **Status** | Select | 상태 (초안, 발행됨, 보관됨) |
| **Content** | Page | 자료의 본문 내용 (페이지 블록) |
| **Summary** | Text | 자료 요약 (목록에서 표시) |
| **Views** | Number | 조회수 |
| **Slug** | Text | URL-friendly 이름 (auto-generated) |
| **Featured** | Checkbox | 홈페이지에 표시할지 여부 |

### Notion 데이터베이스 사용 방법

```
예시 글:
- Title: JavaScript 비동기 처리 완벽 가이드
- Category: JavaScript
- Tags: async/await, Promise, Callback
- Difficulty: 중급
- Published Date: 2026-07-09
- Status: 발행됨
- Content: [Notion 페이지의 모든 콘텐츠]
- Summary: async/await와 Promise의 차이점, 활용법, 실전 예제
```

## 🎨 화면 구성 (UI/UX)

### 1. 홈 페이지 (/)
- **헤더**: 로고, 네비게이션, 검색 바
- **히어로 섹션**: 프로젝트 설명 및 CTA 버튼
- **피처 섹션**: 
  - 최근 발행된 자료 (6-8개)
  - 인기 자료 (카드 형식)
  - 카테고리 바로가기
- **푸터**: 링크, 소셜 미디어

### 2. 자료 목록 페이지 (/notes 또는 /articles)
- **필터 사이드바**:
  - 카테고리 (체크박스)
  - 난이도 (라디오 버튼)
  - 시간 필터 (최신, 인기 등)
- **메인 콘텐츠**:
  - 자료 카드 그리드 또는 리스트
  - 각 카드에 제목, 카테고리, 태그, 발행일, 요약 표시
  - 페이지네이션
- **검색 바**: 상단의 전역 검색

### 3. 자료 상세 페이지 (/notes/[slug])
- **헤더**:
  - 제목
  - 메타데이터 (작성일, 업데이트일, 조회수)
  - 카테고리, 태그 배지
  - 난이도 표시
- **목차 (TOC)**:
  - 우측 사이드바 또는 모바일에서 토글로 표시
  - 헤딩 기반 자동 생성
  - Smooth scroll 지원
- **본문**:
  - Notion 콘텐츠를 HTML로 렌더링
  - 코드 블록 문법 강조
  - 이미지, 테이블, 인용문 등 다양한 형식 지원
- **우측 사이드바** (데스크톱):
  - 관련 글 추천
  - 목차
- **하단**:
  - 공유 버튼 (Twitter, LinkedIn, Copy Link)
  - 이전/다음 글 네비게이션
  - 댓글 섹션 (선택 사항)

### 4. 카테고리 페이지 (/category/[category])
- 특정 카테고리의 모든 자료 표시
- 자료 목록과 유사한 구조
- 카테고리 설명 및 통계

### 5. 검색 결과 페이지 (/search?q=keyword)
- 검색어와 매칭되는 자료 목록
- 검색어 하이라이팅
- 결과 수 표시
- 필터 옵션

## 📝 사용자 입장에서의 플로우

### Content Creator (작성자)
```
1. Notion에서 새 글 작성 (Title, Category, Tags, Content 입력)
2. Status를 "발행됨"으로 변경
3. 약 1-5분 후 웹사이트에 자동으로 반영
4. 수정 후 다시 Notion에서 업데이트하면 자동 동기화
```

### End User (독자)
```
1. 홈페이지 방문
2. 최근 글 목록 확인 또는 검색/필터로 원하는 글 찾기
3. 글 클릭해서 상세 내용 읽기
4. 관련 글 추천받기
5. SNS에 공유하거나 북마크
```

## ⚠️ 제약사항 및 한계

- Notion API의 속도는 데이터베이스 크기에 따라 달라질 수 있음
- 매우 긴 페이지의 경우 렌더링 시간 증가
- 실시간 동기화가 아닌 약간의 지연 가능 (ISR 또는 캐싱)
- Notion API 변경시 코드 수정 필요
